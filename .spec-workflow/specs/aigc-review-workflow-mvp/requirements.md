# AIGC短剧生产线（MVP）— 审核工作流 · 需求规格

## 文档信息

| 项 | 值 |
|----|-----|
| 产品名称 | AIGC短剧生产线（MVP）— 审核工作流 |
| 版本 | V1.1（对齐 Pipeline v2.2） |
| 范围 | 4 个审核节点 + 3 类角色 + 核心页面 |
| 核心目标 | 90% 自动化；人只负责审核、给建议、打回；系统自动定位问题环节并回炉重做 |

---

## 1. 背景与目标

### 1.1 背景

- 上游 Agent 自动完成：剧本理解 → 资产生成 → 关键帧/视频生成 → 视听整合 → 成片合成。
- 人工仅介入 4 个审核节点，形成「自动生产 — 人审 — 打回 — 自动修复 — 再审」闭环。

### 1.2 MVP 必须达成的目标

1. **任务无需人挑选**：进入首页直接看到「已排好优先级的任务面板」，点「处理/继续」进入对应审核页。
2. **每个节点都能打回**：打回必须能关联到：集/镜头/时间戳/资产类别。
3. **系统自动归因**：审核意见提交后，系统自动判断应回到哪个环节（1～4）重做（调接口即可）。
4. **版本可追溯**：每次修改都有「修订总结」与历史版本可回放对比。
5. **多人协同不冲突**：同一集的不同节点可并行；同一节点的任务有锁定机制。
6. **审核员是教练而非操作员**：审核员只做看片与自然语言批注，不承担写 prompt、调模型参数、手动生成等技术操作。

### 1.3 与 core orchestrator 的边界

1. 本 spec 仅定义审核人员工作流、审核页面、审核交互、审核 DTO/API 消费口径。
2. 编排、运行态、回炉、版本、成本、质量、调度等核心业务真相，统一以 `aigc-core-orchestrator-platform` 为准。
3. 本 spec 中出现的任务态、版本态、阶段态，若与 core spec 冲突，必须以 core spec 派生 DTO/API 的方式收口，而不是在本 spec 中另行定义核心真相。

## 2. 角色与职责（RACI）

### 2.1 角色定义

| 角色 | 参与节点 | 权限 |
|------|----------|------|
| **质检员** | 2（视觉素材）、3（视听整合）、4（成片合成） | 可通过/打回；节点3可在时间轴上轻微调整；不可做深度剪辑 |
| **剪辑中台** | 1（美术资产）、4（成片合成） | 节点1：选择定稿资产 + 确认配音音色；节点4：终审；可覆盖质检员结论（可选） |
| **合作方（外部）** | 4（成片合成） | 只读播放 + 提审阅意见 + 通过/不通过；不可见内部资产细节与模型信息（脱敏） |

### 2.2 权限矩阵（MVP）

- 权限矩阵以飞书/内部文档为准；实现时需按角色限制可见范围与操作（通过/打回/定稿/覆盖等）。

## 3. 工作流总览（4 节点）

| 节点 | 名称 | 主要角色 | 审核对象 | 通过后 |
|------|------|----------|----------|--------|
| 1 | 美术资产审核 | 剪辑中台 | 影调/人物/场景/道具 + 配音音色（N07b 生成的候选） | 进入节点2自动生成 |
| 2 | 视觉素材审核 | 质检员 | 关键帧 + 视频片段（含 N16b 节奏/影调调整结果） | 进入节点3 |
| 3 | 视听整合审核 | 质检员 | 多轨时间轴（配音/音效/音乐） | 进入节点4 |
| 4 | 成片合成审核 | 质检员 + 剪辑中台 + 合作方 | 成片逐集验收 | 进入最终通过池 |

打回时：只回炉对应节点，不进入下一节点；由系统/接口做归因与回炉分发。

## 4. 用户故事与需求（EARS）

### 4.1 首页与任务面板

- **US-1**  
  **WHEN** 用户打开首页 **THE SYSTEM SHALL** 展示已按优先级排好的任务面板，且 **SHALL** 提供「处理」/「继续」入口进入对应审核页。

- **US-2**  
  **WHEN** 用户查看任务面板 **THE SYSTEM SHALL** 在左侧展示任务分组：紧急驳回、质检中、新剧待检、Agent 生成中（只读）。

- **US-3**  
  **WHEN** 用户查看任务面板 **THE SYSTEM SHALL** 在顶部展示全局摘要：待审剧数量、待审集数、今日截止/超时数量、生成中数量。

- **US-4**  
  **WHEN** 用户查看任务列表 **THE SYSTEM SHALL** 为每条任务卡展示：剧名、当前节点标签、日期/集数/预计时长、三条进度条（质检员 x/y、中台 x/y、合作方 x/y）、右侧「处理」或「继续」按钮（紧急为红色「处理」，非紧急为绿色「继续」）。

- **US-5**  
  **WHEN** 系统对任务排序 **THE SYSTEM SHALL** 按以下规则（MVP 可写死）：  
  1）紧急驳回 > 质检中 > 新剧待检 > 生成中；  
  2）截止时间越近越靠前；  
  3）合作方驳回优先于内部驳回；  
  4）同级按剩余待审集数少的优先；  
  5）同级再按创建时间。

### 4.2 节点 1：美术资产审核

- **US-6**  
  **WHEN** 剪辑中台进入节点1 **THE SYSTEM SHALL** 展示流程步进条（美术资产/视觉素材/视听整合/成片合成）、左侧导航（剧本要点、影调与氛围、核心人物、核心场景、核心道具）、中间资产候选网格（每资产 4 版）、右侧自然语言交互区、顶部「全部确认」「返回任务」。

- **US-7**  
  **WHEN** 剪辑中台查看资产 **THE SYSTEM SHALL** 为每个资产展示 4 版候选卡片：缩略图、Agent 分数、「锁定/定稿」「修改」操作，以及「已定稿」状态标记。

- **US-8**  
  **WHEN** 剪辑中台在右侧输入自然语言反馈并发送 **THE SYSTEM SHALL** 生成仅针对当前资产类别的重生成任务，重生成完成后在该资产下追加新一轮 4 版（版本号 r2/r3 等）。

- **US-9**
  **WHEN** 剪辑中台在本节点确认配音 **THE SYSTEM SHALL** 展示 Audio Director Agent（N07b）预生成的每个核心角色的音色候选列表（每角色 2-3 个候选样本）、提供试听与确认，确认后写入「本剧音色基线」（CharacterProfile.voice_config）供后续节点复用。

- **US-10**  
  **WHEN** 剪辑中台点击「全部确认」 **THE SYSTEM SHALL** 校验：每个必选资产有定稿（核心人物至少主角 2 人、核心场景至少 1、核心道具按剧本可允许空、影调必须确认），主角音色已确认；若未满足 **SHALL** 禁止提交并提示缺项列表。

- **US-11**  
  **WHEN** 节点1通过 **THE SYSTEM SHALL** 进入节点2自动生成；**WHEN** 打回 **THE SYSTEM SHALL** 仅回炉节点1，不进入节点2。

### 4.3 节点 2：视觉素材审核

- **US-12**  
  **WHEN** 质检员进入节点2 **THE SYSTEM SHALL** 展示流程步进条、左侧工作区（分镜/镜头卡、图片/视频提示词、新视频素材区、修改建议输入、生成参数面板）、中间主预览、底部时间轴（按分镜块）、右侧评分与候选列表。

- **US-13**  
  **WHEN** 质检员查看本集 **THE SYSTEM SHALL** 展示本剧共 N 分镜/镜头、已通过 x、待修改 y、待审核 z；左侧分镜列表可点击跳转到时间轴对应块。

- **US-14**  
  **WHEN** 质检员查看镜头 **THE SYSTEM SHALL** 展示关键帧候选与视频候选（可多版）、对应关系、候选卡含 Agent 分数与「设为时间轴默认」；默认时间轴使用最高分候选。

- **US-15**  
  **WHEN** 质检员提交修改建议 **THE SYSTEM SHALL** 允许选择作用范围（仅关键帧/仅视频/都修改），提交后创建回炉任务、生成新候选并更新右侧候选列表。

- **US-15b（v2.2 新增 — Review Dispatcher 集成）**
  **WHEN** 质检员提交自然语言批注/修改建议 **THE SYSTEM SHALL** 将批注传递给 Review Dispatcher Agent，由其自动解析意图、拆分为可执行任务并调度对应生产 Agent 执行，生成新候选后更新审核页面。

- **US-16**
  **WHEN** 质检员执行通过/打回 **THE SYSTEM SHALL** 支持镜头级「通过该镜头」「打回该镜头（附原因）」及分集级「本集通过」「驳回修改」；打回进入回炉。

- **US-16b（v2.2 新增 — 快速通过）**
  **WHEN** 质检员查看节点 2 审核页 **THE SYSTEM SHALL** 提供「全部接受 AI 推荐」快速通过按钮，允许一键确认所有 AI 已自动推荐的最高分候选，无需逐 shot 手动确认。点击后系统标记所有 shot 为 approved 并整体放行。

- **US-17**  
  **WHEN** 最高分低于阈值（如 8.0） **THE SYSTEM SHALL** 将镜头标记为「需关注」；质检员仍可通过但须写备注（MVP 可固定阈值）。

### 4.4 节点 3：视听整合审核

- **US-18**  
  **WHEN** 质检员进入节点3 **THE SYSTEM SHALL** 展示左侧素材库（音效/配音/音乐 tab）、中间竖屏预览与多轨时间轴、右侧属性面板（音量、淡入淡出等）、顶部节点状态/导出/本集通过。

- **US-19**  
  **WHEN** 质检员编辑时间轴 **THE SYSTEM SHALL** 至少提供三轨：配音轨（TTS/对白）、音效轨（SFX）、音乐轨（BGM/AI Music）；允许替换素材、微调对齐、调整音量与淡入淡出、删除片段；**SHALL NOT** 提供复杂剪辑/专业调音（MVP 不做或权限禁止）。

- **US-19A（时间轴拖拽与裁切边界）**  
  **WHEN** 用户执行节点3时间轴编辑 **THE SYSTEM SHALL** 仅允许轻量编辑命令：`move_clip / trim_clip / replace_clip / set_volume / set_fade / delete_clip`；并且必须满足：  
  1）`start_sec >= 0`；2）`end_sec > start_sec`；3）`volume` 范围 `0.0~2.0`；4）默认不允许跨轨迁移；5）默认不允许生成负时长或越界片段。

- **US-19B（持久化与回显）**  
  **WHEN** 用户保存节点3时间轴编辑 **THE SYSTEM SHALL** 将变更持久化到时间轴片段结构（如 `timeline_clips`）并返回最新时间轴快照；刷新页面后编辑结果必须可回显且不丢失。

- **US-19C（并发冲突控制）**  
  **WHEN** 两个会话同时编辑同一时间轴 **THE SYSTEM SHALL** 基于 `timeline_revision` 做乐观并发控制；版本不一致时返回冲突（如 409）并附最新快照摘要，禁止静默覆盖。

- **US-19D（审计与归因）**  
  **WHEN** 时间轴编辑落库 **THE SYSTEM SHALL** 记录操作审计（操作者、命令类型、前后差异、时间、任务/版本锚点），并可追溯到对应审核任务与版本。

- **US-20**  
  **WHEN** 存在 BGM 盖过配音、轨道空洞、配音与字幕不同步等情况 **THE SYSTEM SHALL** 给出相应自动提示（如「建议降低音乐音量」「此处无环境音」「需回到成片阶段处理」）。

- **US-21**  
  **WHEN** 质检员打回 **THE SYSTEM SHALL** 要求选择问题归属（视觉素材/视听整合/其他）且打回理由必填。

### 4.5 节点 4：成片合成审核

- **US-22**  
  **WHEN** 审核方进入节点4 **THE SYSTEM SHALL** 展示中间播放器、左侧修订记录与历史版本卡片、右侧带时间戳的审阅意见与问题归属、「添加审阅打点」、底部剧集列表与审核链路状态（质检员/中台/合作方）。

- **US-23**  
  **WHEN** 审核方添加审阅意见 **THE SYSTEM SHALL** 使用播放器当前时间作为审阅时间点，允许输入说明、选择问题归属（节点2/3/4或其他），并生成可点击跳转的打点列表。

- **US-24**  
  **WHEN** 审核方点击「驳回修改」 **THE SYSTEM SHALL** 要求至少 1 个打点或一段文字说明，并根据归属将任务退回对应节点，该集进入「生成中」且旧版只读锁定。

- **US-25**  
  **WHEN** 审核方查看左侧 **THE SYSTEM SHALL** 展示修订记录（Agent 修订点列表）及历史版本卡（可应用回滚或对比）。

- **US-26**  
  **WHEN** 多方审核 **THE SYSTEM SHALL** 支持顺序：质检员通过 → 待中台审核；中台通过 → 待合作方审核；合作方通过 → 最终通过池（MVP 建议顺序执行）。

- **US-27**  
  **WHEN** 合作方查看 **THE SYSTEM SHALL** 仅展示待其审核的集、脱敏后成片与审阅能力，不展示内部资产与模型信息。

### 4.6 通用系统能力

- **US-28**  
  **WHEN** 用户点击「处理/继续」 **THE SYSTEM SHALL** 将任务锁定（in_progress），他人进入仅只读并提示「由 XX 处理中」；超时（如 30 分钟，可配置）未操作 **SHALL** 自动释放锁。

- **US-28B（v2.2 新增 — OpenClaw 任务入口一致性）**  
  **WHEN** 审核任务由 OpenClaw 创建或更新 **THE SYSTEM SHALL** 通过统一 Review Gateway DTO 在任务面板与节点页实时反映任务状态，避免页面层直接依赖底层表结构字段。

- **US-29**  
  **WHEN** 任意节点产生打回与重生成 **THE SYSTEM SHALL** 生成新版本号（v1/v2/v3…），且任意版本可回放（至少成片阶段）；版本与审核意见强绑定。

- **US-30**  
  **WHEN** 用户在任意节点输入自然语言反馈 **THE SYSTEM SHALL** 落库为结构化字段：scope（人物/场景/道具/影调/关键帧/视频/配音/音效/音乐/字幕/合成）、severity（blocker/major/minor）、anchor（asset_id/shot_id/timestamp）、note（自由文本）。

- **US-31**  
  **WHEN** 节点4打回 **THE SYSTEM SHALL** 由人选归属；节点2/3打回默认归属本节点，但允许上游归因（如视觉问题回到节点1或2）。

## 5. 数据对象（MVP 最小集合）

> 说明：
> 本节描述审核工作流页面所需的消费层对象与最小持久化集合，不等同于系统级核心业务真相源。
> 核心真相对象（如 `EpisodeVersion`、`Run`、`NodeRun`、`ReviewTask`、`ReturnTicket` 等）以 core spec 为准。

- **Project / Series**：项目/剧集系列  
- **Episode**：集  
- **StageTask**：节点任务（1～4）  
- **Asset**：美术资产（人物/场景/道具/影调）  
- **Variant**：候选版本（4 版、多版）  
- **Shot**：分镜/镜头  
- **Timeline**：时间轴（节点2/3）  
- **ReviewPoint**：审阅打点（timecode + note + stage）  
- **ReviewDecision**：通过/驳回 + 备注  
- **RevisionLog**：修订记录（Agent summary）  
- **AuditLog**：操作审计

## 6. 埋点与统计（MVP 必须）

- **任务维度**：领取/开始/完成时间，打回次数，每节点平均耗时。  
- **内容质量维度**：通过率（按节点、按人），Top 问题分类（节点4最重要），同一问题重复出现率。  
- **交付维度**：每日通过集数，合作方驳回率，回炉平均轮次。  

> 边界说明：
> 本节统计主要服务审核流程页面与审核运营展示；成本红线、系统吞吐、质量总评、Reflection 沉淀率等系统级指标由 core spec 负责定义与计算。

## 7. 验收标准（DoD）

### 节点 1（美术资产）

- 能看到 4 版候选 + 分数；能锁定定稿；不满意能自然语言反馈并触发新一轮候选；能确认主角音色。

### 节点 2（视觉素材）

- 每集按时间轴排布；默认最高分上轨；可查看多候选并切换上轨；可通过/打回并提交建议。

### 节点 3（视听整合）

- 多轨道可播放；支持替换素材与微调；支持音量/淡入淡出；可通过/打回。
- 节点3时间轴编辑后刷新页面，参数（起止、音量、淡入淡出、替换结果）必须保持一致。
- 并发编辑发生冲突时，必须提示冲突并提供最新版本快照，不得静默覆盖。

### 节点 4（成片合成）

- 逐集播放验收；时间戳打点可回放；打回能选择归属节点并触发回炉；左侧能看到修订记录与历史版本。

## 8. 核心页面映射约束（基于现有设计稿）

本节用于将已提供的核心页面设计转成可执行产品约束，避免开发偏离视觉与交互。

### 8.1 任务首页（我的任务）

- 页面必须包含顶部统计区，至少展示：待评审总量、已驳回数量、生成中数量。
- 左侧必须包含任务分组导航，且每组显示数量徽标。
- 中间任务卡片必须包含：剧名、集数/时长、节点状态标签、三条角色进度条、主操作按钮（处理/继续）。
- 任务卡必须支持紧急视觉标识（如红色边框/红色按钮）与普通任务视觉区分。
- 任务卡点击后必须可直接进入对应集与节点，不额外弹中间确认。

### 8.2 节点1（美术资产审核）

- 顶部必须展示四阶段流程指示器，当前阶段高亮。
- 左侧必须保留资产类别导航（剧本要点、影调、人物、场景、道具）。
- 中间必须为候选网格布局，支持 4 版并列展示，卡片内显示评分与「锁定/修改」操作。
- 右侧必须保留自然语言反馈区，支持在当前资产上下文内发起重生成。
- 顶部操作区必须包含「返回任务」与「全部确认」。

### 8.3 节点2（视觉素材审核）

- 页面必须采用三栏结构：左侧建议与参数、中间主预览、右侧候选与评分。
- 底部必须提供分镜时间轴，按镜头块展示当前默认版本，支持切换与定位。
- 左侧修改建议提交时必须能绑定当前镜头与作用范围（关键帧/视频/都修改）。
- 右侧候选列表必须显示分数与版本信息，并支持「设为默认」。
- 节点级操作区必须包含「驳回修改」与「本集通过」。

### 8.4 节点3（视听整合审核）

- 左侧必须为素材库（音效/配音/音乐）并支持搜索与筛选。
- 中间必须为竖屏播放器 + 多轨时间轴，至少三轨：配音、音效、音乐。
- 右侧属性面板必须支持音量、淡入、淡出参数编辑。
- 轨道片段必须支持拖拽微调与替换，不提供复杂剪辑能力。
- 页面必须提供「本集通过」入口，且打回需填写归属与理由。
- 时间轴保存必须返回并展示 `timeline_revision`（或等价版本号），用于冲突检测与重试提示。

### 8.5 节点4（成片合成审核）

- 左侧必须展示修订记录时间线与历史版本卡（可回滚/对比）。
- 中间必须展示成片播放器，底部展示分集状态卡。
- 右侧必须提供时间戳审阅输入、归属选择、添加打点功能。
- 驳回前必须有至少一个审阅打点或文字说明。
- 审核链路状态必须可见（质检员/中台/合作方），并体现当前卡点。

### 8.6 统一交互约束

- 所有「通过/驳回」操作均需写入 AuditLog。
- 所有打回都必须带 anchor（asset_id/shot_id/timecode 之一）。
- 所有版本切换必须可追溯到 RevisionLog 与对应 ReviewDecision。
